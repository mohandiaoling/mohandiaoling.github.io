博客内容创建

  1. 创建一个博客网站：hugo new site myblog（站点名，可自定义）
  2. 创建章节：hugo new --kind chapter <name>/_index.md
  3. 创建章节下页面：hugo new <name>/name.md 若要该页面显示，需删除draft: true
  4. 启动站点服务：hugo server

部署到github

  1. 在博客目录下打开cmd，输入hugo --theme=hugo-theme-learn --baseUrl="https://mohandiaoling.github.io" --buildDrafts
    会在博客根目录 myblog 生成 public 文件夹，里面就是编译好的静态页面文件。
  2. 在public目录下：运行git bash，执行 git init将public文件夹变成 git 本地仓库
  3. 执行git add .将文件夹所以文件放入暂存区
  4. 执行 git commit -m "提交文件" 提交暂存区到本地仓库中
  5. 执行git remote add origin git@github.com:mohandiaoling/mohandiaoling.github.io.git添加远程版本库
  6. 然后执行git pull
  7. 执行git push -u origin master -f将本地仓库代码推送到远程库
  8. 执行git push -u origin master,直到出现Everything up-to-date。
  9. 最后，在浏览器中访问 https://mohandiaoling.github.io/ ，即可看到自己的博客网站。

文本内容编辑

  • 标题：按#，从一到六，标题从大到小，一个#号时，标题位于中间。并且#与后面文档有一个空格。例如:
# h1 Heading
## h2 Heading
### h3 Heading
#### h4 Heading
##### h5 Heading
###### h6 Heading

效果为：

h1 Heading

h4 Heading

h5 Heading
h6 Heading
  • 注释：<!-- 注释内容 -->与html相同
  • 主题分隔：（1）三个连续的下划线 （2）三个连续的破折号 （3）三个连续的星号。这三个显示是一样的。例如：___,***.---.
    效果为：
  • 代码突出显示：在代码俩边分别加上```（英文下）。
  • 页面换行：在一行后面加俩个空格，再按回车
  • 粗体文本：在文本俩侧各加上俩个**，例如：**rendered as bold text**
    效果为：rendered as bold text
  • 斜体文本：在文本俩侧各加上一个下划线，例如：**rendered as bold text**
    效果为：rendered as italicized text
  • 删除线：文本俩侧各加上俩个波浪线,例如：~~Strike through this text.~~
    效果为：Strike through this text.
  • 引用文档中其他来源的内容块：在要引用的任何文本之前添加>。注意>需要放在一行的最前面。例如：
> **Fusion Drive** combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.

效果为：

Fusion Drive combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.
也可用于嵌套：例如

> **Fusion Drive** combines a hard drive with a flash storage (solid-state drive) and presents it as a single logical volume with the space of both drives combined.

效果为：

+ valid bullet

  • valid bullet
  • valid bullet

  • 表格：通过将管道添加为每个单元格之间的分隔线，并在标题下添加一行短划线（也由条形分隔）来创建表。请注意，管道不需要垂直对齐。例如：
| Option | Description |
| ------ | ----------- |
| data   | path to data files to supply the data that will be passed into templates. |
| engine | engine to be used for processing templates. Handlebars is the default. |
| ext    | extension to be used for dest files. |

效果为：

Option Description
data path to data files to supply the data that will be passed into templates.
engine engine to be used for processing templates. Handlebars is the default.
ext extension to be used for dest files.
  • 表格右对齐：在任何标题下方的破折号右侧添加冒号将使该列的文本右对齐。例如：
| Option | Description |
| ------:| -----------:|
| data   | path to data files to supply the data that will be passed into templates. |
| engine | engine to be used for processing templates. Handlebars is the default. |
| ext    | extension to be used for dest files. |

效果为：

Option Description
data path to data files to supply the data that will be passed into templates.
engine engine to be used for processing templates. Handlebars is the default.
ext extension to be used for dest files.
  • 链接：将鼠标悬停在链接上，没有工具提示。例如：[Assemble](http://assemble.io) 效果为：Assemble
  • 添加工具提示：将鼠标悬停在链接上，应该有一个工具提示。与链接格式不同的是，后面“”里面有工具提示内容。例如：
[Upstage](https://github.com/upstage/ "Visit Upstage!")

效果为：Upstage

  • 图像：图像具有与链接类似的语法，但包含前面的感叹号。例如： ![Minion](https://octodex.github.com/images/minion.png)
    效果为：
  • 图像的替代用法：例如：![Alt text][id]然后在文档后面加上[id]: https://octodex.github.com/images/dojocat.jpg "The Dojocat"
  • 调整图片大小：将 HTTP 参数和/或添加到链接图像以调整图像大小。值是 CSS 值（默认值为 ）。例如：![Minion](https://octodex.github.com/images/minion.png?height=50px&width=300px) 效果为：
